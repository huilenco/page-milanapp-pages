# page-milanapp-pages

## 项目简介

本仓库用于归档和发布多个独立 HTML 页面。这些页面通常用于展示、测试或作为静态资源使用，不针对任何特定域名或具体网站。

该仓库的主要目的是提供一种简单、可维护的方式来托管零散的 HTML 文件，便于版本管理和历史回顾。

## 目录说明

```
.
├── index.html          # 仓库页面入口（可选）
├── pages/              # 存放归档的 HTML 页面
│   ├── example-page/   # 单个页面示例目录
│   │   └── index.html
│   └── ...
├── assets/             # 公共资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- **pages/**：每个子目录对应一个独立的 HTML 页面，目录名建议使用简短有意义的英文或拼音。
- **assets/**：存放页面间共享的静态资源，避免重复引用。
- **index.html**：可选的顶层入口文件，用于快速导航到各个归档页面。

## 页面归档说明

- 所有页面均为独立 HTML 文件，不依赖后端服务或数据库。
- 页面之间无强制关联，但可共享 `assets/` 中的公共资源。
- 归档的页面通常保持稳定，不频繁修改，以保留历史版本。

## 维护说明

- 新增页面：在 `pages/` 下创建新目录，放入 `index.html` 及相关资源。
- 更新页面：直接修改对应文件，建议在 commit 信息中说明变更原因。
- 删除页面：移除对应目录，并检查是否有其他页面引用其资源。
- 资源管理：公共资源放入 `assets/`，避免在每个页面目录中重复存放。

## 使用方式

你可以通过 GitHub Pages 或其他静态托管服务部署本仓库中的页面。部署后，每个页面的访问路径通常为：

```
https://<你的域名>/pages/<页面目录名>/
```

或直接访问 `pages/` 下的 `index.html` 文件。
